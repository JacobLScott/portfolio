UE5 Level Design

Personal Project • Level Design • Unreal Engine 5

Project Overview

Role Solo Level Designer
Team Size Solo
Timeframe January 2026 – Present
Engine / Tools Unreal Engine 5

This project highlights the iterative pipeline I use to prototype 3D environments in Unreal Engine 5 — from initial concept sketches through blockout and into refined blockmesh. The goal is to rapidly test and communicate spatial ideas before committing time to full asset production.

Design Goals

Each environment should communicate a clear player flow and a sense of place using only primitive geometry. The process prioritizes speed of iteration: a layout should be testable within a session and improvable based on what that test reveals. Visual polish is intentionally deferred until the core spatial experience is validated.

Highlights

  • Sketched and refined environment concepts before touching the engine
  • Translated 2D sketches into rapid 3D blockouts in UE5
  • Iterated each blockout into a blockmesh pass with more architectural detail
  • Tested player movement, sightlines, and scale at each stage

Design Process

1. Conceptualization

The process begins with quick sketches and rough top-down layouts. These establish the core flow — major pathways, focal points, chokepoints — before any engine work begins. Sketching is cheap; it is the fastest way to explore and discard ideas.

2. Initial Blockout

Using basic geometric shapes in UE5, I build a rough 3D representation of the environment focusing on:

  • Establishing correct scale and proportions relative to a character capsule
  • Defining major pathways and intended sightlines
  • Verifying basic player movement and navigation feel

3. Blockmesh Refinement

The blockout is refined into a blockmesh — still abstract, but with more defined surfaces:

  • Architectural details that communicate function (stairs, walls, cover)
  • Preliminary lighting to test atmosphere and readability
  • Playtesting to confirm spatial flow before any assets are placed

4. Iteration

Each stage feeds back into the previous one. Playtesting a blockout often reveals a layout issue that is much cheaper to fix at the sketch stage. This loop — sketch, build, test, revise — is the core of the process.
